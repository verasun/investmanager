📋 系统概述

InvestManager 是一个功能完整的量化投资分析系统，支持 A 股、美股数据采集、技术分析、策略回测和报告生成。v2.0 版本采用单容器架构，大幅简化了部署流程。

🗄️

SQLite 数据库

嵌入式数据库，无需外部依赖，数据持久化存储，支持时序数据优化

⚡

内存缓存

内置 LRU 缓存替代 Redis，支持 TTL 过期策略，零外部依赖

🔐

OAuth2 认证

支持 Gmail、Outlook OAuth2 安全认证，无需存储邮箱密码

💬

飞书集成

支持机器人交互、多维表格触发、消息推送和文档生成

核心功能

  • 多数据源支持：AkShare (A股)、yfinance (美股)、Tushare
  • 技术指标计算：MA、MACD、RSI、布林带等 20+ 指标
  • 策略回测：动量策略、均值回归、趋势跟踪
  • 风险管理：仓位控制、敞口监控、预警通知
  • 自动报告：日报、周报、月报自动生成与发送
  • 任务调度：支持定时触发和事件触发

⚙️ 配置说明

环境变量配置

系统支持通过环境变量或 .env 文件进行配置。创建 .env 文件：

.env 配置文件 
# ============================================
# InvestManager 环境配置
# ============================================

# 应用配置
APP_ENV=production
LOG_LEVEL=INFO

# 数据库配置 (SQLite)
DATABASE_BACKEND=sqlite
SQLITE_DB_PATH=/app/data/investmanager.db

# 本地缓存配置
LOCAL_CACHE_ENABLED=true
LOCAL_CACHE_MAX_SIZE=1000
LOCAL_CACHE_TTL=300

# API Keys
TUSHARE_TOKEN=your_tushare_token
OPENAI_API_KEY=your_openai_key
ANTHROPIC_API_KEY=your_anthropic_key

# 调度器配置
SCHEDULER_ENABLED=true
DAILY_ANALYSIS_TIME=18:00

配置参数详解

参数名 默认值 说明
DATABASE_BACKEND sqlite 数据库类型：sqlite 或 postgresql
SQLITE_DB_PATH ./data/investmanager.db SQLite 数据库文件路径
LOCAL_CACHE_MAX_SIZE 1000 缓存最大条目数
LOCAL_CACHE_TTL 300 缓存过期时间（秒）
SCHEDULER_ENABLED true 是否启用定时任务
DAILY_ANALYSIS_TIME 18:00 每日分析执行时间

🚀 部署指南

快速部署（推荐）

克隆项目

Terminal 
git clone https://github.com/your-org/investmanager.git
cd investmanager

创建配置文件

Terminal 
cp .env.example .env
# 编辑 .env 文件，填入你的配置

启动服务

Terminal 
docker-compose -f docker-compose.standalone.yml up -d

验证部署

Terminal 
# 检查服务状态
docker-compose -f docker-compose.standalone.yml ps

# 健康检查
curl http://localhost:8000/health

# 查看日志
docker-compose -f docker-compose.standalone.yml logs -f

Docker Compose 配置

docker-compose.standalone.yml 
version: '3.8'

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile.standalone
    image: investmanager:standalone
    container_name: investmanager
    ports:
      - "8000:8000"   # API
      - "8501:8501"   # Web UI (可选)
    environment:
      - APP_ENV=${APP_ENV:-production}
      - DATABASE_BACKEND=sqlite
      - SQLITE_DB_PATH=/app/data/investmanager.db
      # 从 .env 文件加载其他变量
    volumes:
      - investmanager-data:/app/data
      - investmanager-reports:/app/reports
      - investmanager-logs:/app/logs
    restart: unless-stopped

volumes:
  investmanager-data:
  investmanager-reports:
  investmanager-logs:

生产环境部署

⚠️
生产环境建议

建议配置 Nginx 反向代理、HTTPS 证书、日志轮转和监控告警。

Nginx 反向代理配置 
server {
    listen 80;
    server_name invest.yourdomain.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name invest.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 飞书 Webhook
    location /api/feishu/ {
        proxy_pass http://localhost:8000/api/feishu/;
        proxy_read_timeout 60s;
    }
}

📧 邮件配置

OAuth2 认证配置（推荐）

Gmail OAuth2 设置步骤

创建 Google Cloud 项目

访问 Google Cloud Console，创建新项目

启用 Gmail API

在 API 库中搜索并启用 Gmail API

创建 OAuth 凭据

在「凭据」中创建 OAuth 客户端 ID，选择「桌面应用」

运行授权脚本

Terminal 
python scripts/email_oauth2_setup.py \
    --provider gmail \
    --client-id "YOUR_CLIENT_ID" \
    --client-secret "YOUR_CLIENT_SECRET"

Outlook OAuth2 设置步骤

注册 Azure 应用

访问 Azure Portal，注册新应用

配置权限

添加 Microsoft Graph 权限：SMTP.Send 和 offline_access

创建客户端密钥

在「证书和密钥」中创建新的客户端密钥

运行授权脚本

Terminal 
python scripts/email_oauth2_setup.py \
    --provider outlook \
    --client-id "YOUR_CLIENT_ID" \
    --client-secret "YOUR_CLIENT_SECRET"

OAuth2 优势

相比密码认证，OAuth2 更安全：无需存储邮箱密码、令牌可随时撤销、支持双因素认证。

数据备份

备份脚本 
#!/bin/bash
# backup.sh - 数据备份脚本

BACKUP_DIR=/backup/investmanager
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份 SQLite 数据库
docker cp investmanager:/app/data/investmanager.db \
    $BACKUP_DIR/investmanager_$DATE.db

# 备份报告文件
docker cp investmanager:/app/reports \
    $BACKUP_DIR/reports_$DATE

# 压缩备份
tar -czf $BACKUP_DIR/backup_$DATE.tar.gz \
    $BACKUP_DIR/investmanager_$DATE.db \
    $BACKUP_DIR/reports_$DATE

# 清理临时文件
rm -rf $BACKUP_DIR/investmanager_$DATE.db \
       $BACKUP_DIR/reports_$DATE

# 保留最近 7 天的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +7 -delete

echo "Backup completed: backup_$DATE.tar.gz"

🔍 故障排除

常见问题

❌ 容器无法启动

症状：容器启动后立即退出

排查：

docker logs investmanager

检查端口是否被占用、配置文件是否存在

❌ 数据库连接失败

症状：日志显示 "database is locked"

解决：

# 检查数据目录权限
docker exec investmanager ls -la /app/data

# 重建数据目录
docker-compose down -v
docker-compose up -d

❌ 飞书消息收不到

症状：机器人无响应

排查：

  1. 检查 Webhook URL 是否正确
  2. 验证 Token 是否匹配
  3. 查看容器日志确认事件是否到达

❌ 邮件发送失败

症状：报告未发送

排查：

# 测试邮件发送
python scripts/email_oauth2_setup.py \
    --test your@email.com

日志级别调整

调整日志级别 
# 在 .env 中设置
LOG_LEVEL=DEBUG  # DEBUG, INFO, WARNING, ERROR

# 重启服务生效
docker-compose -f docker-compose.standalone.yml restart

性能优化建议

  • 定期清理历史数据，保持数据库在合理大小
  • 根据监控数据调整缓存大小 (LOCAL_CACHE_MAX_SIZE)
  • 使用 Nginx 静态文件缓存减轻 API 压力
  • 对于高频查询，考虑添加数据库索引
