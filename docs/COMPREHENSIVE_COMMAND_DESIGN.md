# 组合指令模式设计方案

## 概述

新增组合指令模式，用户只需一句话即可串行执行完整分析流程，直接返回最终综合结果。现有分阶段模式保持不变。

---

## 意图类型对比

| 意图 | 触发词示例 | 执行流程 | 说明 |
|------|-----------|---------|------|
| analyze | "分析601688" | 技术分析 | 快速分析，仅返回技术指标 |
| backtest | "回测均线策略" | 策略回测 | 单独回测，返回收益统计 |
| **comprehensive** (新) | "综合分析601688" | 数据获取→技术分析→策略回测 | 完整流程，返回综合报告 |

---

## Comprehensive 执行流程

```
用户输入: "综合分析601688"

Step 1: 数据获取
├── 获取30天历史K线数据
├── 获取实时行情
└── 获取股票基本信息

Step 2: 技术分析
├── 计算均线 (MA5/MA10/MA20)
├── 计算MACD
├── 计算RSI
├── 判断趋势方向
└── 识别支撑/压力位

Step 3: 策略回测
├── 执行均线策略回测
├── 计算收益率
├── 计算最大回撤
└── 统计交易次数/胜率

Step 4: 生成报告
└── 返回综合分析报告
```

## 实现要点

### 1. 新增意图类型

**文件**: `src/feishu/bot.py`

```python
class CommandType(Enum):
    ...
    COMPREHENSIVE = "comprehensive"  # 组合指令
```

### 2. LLM 意图识别

**文件**: `src/feishu/intent_parser.py`

更新 SYSTEM_PROMPT 添加 `comprehensive` 意图类型：
- 触发词: "综合分析"、"完整分析"、"深度评估"、"全部分析"

### 3. 组合指令处理器

**文件**: `src/feishu/handlers.py`

新增 `handle_comprehensive` 函数：
1. 调用数据获取逻辑
2. 调用技术分析逻辑
3. 调用策略回测逻辑
4. 汇总生成综合报告
5. 发送最终结果

### 4. 异步执行优化

- 使用 `asyncio.gather` 并行获取数据
- 超时处理：单个步骤超时不影响整体
- 进度提示：可选发送"正在分析中..."提示
